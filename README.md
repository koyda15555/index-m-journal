# index-m-journal

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为页面内容的结构化存档与展示空间。  
所有 HTML 文件均以独立页面形式存在，可被直接访问或引用。

## 目录说明

```
index-m-journal/
├── index.html          # 仓库主入口页（可选）
├── page/               # 独立页面存放目录
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 共用静态资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本说明文件
```

- `page/` 目录用于存放各个独立的 HTML 页面，每个页面可包含完整的内容与样式。
- `assets/` 目录用于存放多个页面可能共用的资源文件，如 CSS、JavaScript 或图片。
- `index.html` 为可选的主入口页面，可根据需要创建。

## 页面归档说明

本仓库中的 HTML 页面均以独立文档形式存在，适合以下使用场景：

- 备份或归档特定内容的 HTML 快照。
- 展示不依赖后端服务的静态页面集合。
- 作为个人或项目的内容记录与索引。

每个页面应尽量保持自包含，资源引用建议使用相对路径，以确保在本地或 GitHub Pages 下均可正常访问。

## 维护说明

- 新增页面：在 `page/` 目录下创建新的 HTML 文件，并确保引用资源路径正确。
- 更新页面：直接修改对应 HTML 文件，并同步更新相关资源。
- 删除页面：移除对应 HTML 文件及其引用的专用资源。
- 建议为每个页面添加简单的头部注释，标明创建日期或版本信息。
